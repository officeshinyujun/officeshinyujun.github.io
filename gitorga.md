git명령어 종류
---
- committ
* 의미있는 변경 작업들을 로컬 작업소에 저장하는 동작
- push
* 로컬 작업소에 저장한 내용을 원격 저장소로 전송하는 명령어
- pull
* 원격 저장소에 있는 프로젝트의 변경사항을 그대로 로컬저장소에 옮겨와 자동 병합하는것, 반대도 ㄱㄴ
- merge
* 서로 다른 브랜치에서 작업을 하고 원격 저장소에 올려놓았을때, 합쳐서 한번에 변경 사항으로 원격 저장소에 올리는것
- checkout
* 로컬 저장소에서 브런치 바꾸는 용도
- init
* 새로운 깃 저장소를 생성할때 사용
- rebase
* 머지랑 다르게 개별로 원격 저장소에 합칠 수 있다 예로들어 a,b 브런치가 있을때, 만약 a가 작업이 덜 됐고 b가 다 되었다면 b먼저 리베이스 가능한 것
- branch
* 깃의 커밋 사이를 가볍게 이동 가능한 포인터
